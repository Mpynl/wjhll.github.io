十一、Linux文件管理命令mv、rm、which、tree命令

移动和重命名文件

  mv [OPTION]...[-T] SOURCE DEST

  mv [OPTION]...SOURCE...DIRECTORY

  mv [OPTION]...-t DIRECTORY SOURCE...

    常用选项：

      -i: 交互式

      -f: 强制

删除

  rm [OPTION]…FILE..

  常用选项：

    -i 交互式

    -f 强制删除

    -r 递归

    –no-preserve-root 删除

  示例：

   &emsp  rm –rf /

目录操作

  tree 显示目录树

    -d: 只显示目录

    -L level：指定显示的层级数目

    -P pattern:只显示由指定pattern匹配到的路径

  mkdir 创建目录

    -p: 存在于不报错，且可自动创建所需的各目录

    -v: 显示详细信息

    -m MODE: 创建目录时直接指定权限

  rmdir删除空目录

    -p: 递归删除父空目录

    -v: 显示详细信息

  rm –r 递归删除目录树

索引节点

  inode（index node）表中包含文件系统所有文件列表

  一个节点（索引节点）是在一个表项，包含有关文件的信息（元数据），包括：

    文件类型，权限，UID，GID

    链接数（指向这个文件名路径名称个数）

    该文件的大小和不同的时间戳

    指向磁盘上文件的数据块指针

    有关文件的其他数据



  文件引用一个是inode号

  人是通过文件名来引用一个文件

  一个目录是目录下的文件名和文件inode号之间的映射


在CP的命令:

    分配一个仝闲的inode号，在inode表中生成新条目

    在目录中创建一个目录项，名称与inode编号关联

    拷贝数据生成新的文件

rm 命令

    链接数递减，从而释放的inode号可以被重用

    把数据块放在空闲列表中

    删除目录项

    数据实际上不会马上被删除，但当另一个文件使用数据块时将被覆盖。
